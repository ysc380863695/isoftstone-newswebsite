## 开发指南

### 技术栈
- HTML/CSS/JS + React CDN (Babel standalone)
- **无构建步骤**——所有JS通过 `<script type="text/babel">` 在浏览器端编译
- 设计风格：Pentagram配方（专业杂志风）+ Vercel科技感
- 主色：软通品牌蓝(#004B8D) + 深灰背景(#0A0A0A) + 白色文字(#FFFFFF)
- 辅助色：浅灰(#F5F5F5)、强调橙(#FF6B35)

### TDD 流程
1. 先写测试（RED）→ 2. 跑测试确认失败 → 3. 写最小实现（GREEN）→ 4. 重构（IMPROVE）

### 垂直切片

正确：test1→impl1, test2→impl2, test3→impl3
错误：test1,test2,test3 → impl1,impl2,impl3

### Contract-First

前后端由不同 agent 实现。字段名严格从 `docs/api-contracts.md` 抄写，禁止自己发明。

### Doc-Code Sync

当你变更了前端API调用 → **必须**通知 backend-dev 同步更新 api-contracts.md

### 代码质量
- 函数 <50 行，文件 <800 行
- React组件注意不必要的重渲染，使用 memoization
- 无障碍：ARIA 标签
- 响应式设计：移动端优先
- 3M带宽：gzip压缩，避免大静态资源，图片懒加载

### 前端页面设计要点

1. **首页Hero**：精选3条重要新闻大图轮播
2. **分类标签栏**：8个分类横向滚动筛选
3. **新闻卡片网格**：封面图+标题+摘要+分类+日期+信源
4. **新闻详情**：模态弹层，AI摘要高亮+关键信息侧栏+相关推荐
5. **右下角问答**：小圆点悬浮按钮 → 360px对话窗 → 回复内嵌新闻引用

### 动效
- 卡片悬停3D倾斜
- 滚动视差
- 骨架屏加载
- 分类切换平滑过渡

### 前端先用Mock数据开发

你可以先用Mock数据开发所有组件，不依赖后端。Mock数据格式参考 `docs/api-contracts.md` 中的响应格式。
